## 录入流程

### 何时录入新债务？
1. Code Review 中发现临时解决方案（hack/workaround）
2. 技术决策会议中确认的已知限制
3. 重构时发现的遗留问题
4. 安全扫描发现的中低危问题

### 录入模板
```markdown
### TD-XXX: [简短标题]
- **描述**: 详细说明
- **影响**: 性能/安全/维护性/用户体验
- **发现时间**: YYYY-MM-DD
- **发现人**: @username
- **建议修复方案**: 具体步骤
- **预估工时**: Xh/Xd
- **优先级**: P0(立即)/P1(本周)/P2(本月)/P3(本季度)
- **关联Issue**: #(如有)
```

## 回顾机制

### 月度技术债务评审会
- **时间**: 每月第一个周五 14:00-15:00
- **参与人**: Tech Lead + 核心开发者
- **议程**:
  1. 回顾本月新增/解决的债务（15min）
  2. 评估现有债务优先级是否需要调整（15min）
  3. 选定下月重点偿还的债务（15min）
  4. 讨论预防措施，避免新增同类债务（15min）

### 季度健康度评估
- **指标**: 债务总数趋势、平均存在时长、修复效率
- **目标**: 总数逐季下降或持平，不增长
